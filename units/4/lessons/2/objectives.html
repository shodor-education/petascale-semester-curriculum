By the end of this lesson, students will be able to:

  • Demonstrate an understanding of dynamic programming approach to the Longest Common Sequence problem.
  • Show why filling the cost table by rows (or columns) creates a data dependency.
  • Show how the data dependency was avoided by filling the cost table diagonally.
  • Demonstrate that using OpenMP makes the diagonal cost table filling function parallel.
  • Show how the parallel version executes quicker than the serial version for the larger data set.
