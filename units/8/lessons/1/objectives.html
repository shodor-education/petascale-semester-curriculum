By the end of this lesson, students will be able to:

  • Know what is heterogenous memory/computing with OpenACC and its use cases.
  • Find the compute intensive parts of programs.
  • Understand the two methods for adding compiler directives:
    • One where the program flow and data flow is controlled by the programmer.
    • One where all the work is left for the compiler to decide which loops to parallelize and how to copy data between CPU <=> GPU.
  • Avoid bad practices in OpenACC programming that could lead to potential performance issues.
  • Accelerating scientific applications using PGI compiler.
