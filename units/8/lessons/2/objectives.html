By the end of this module, students will:

  1. Knowledge of what is OpenACC and its user cases
  2. Understand the two methods for adding compiler directives:
    1. One where the program flow and data flow is controlled by the programmer
    2. One where all the work is left for the compiler to decide which loops to parallelize and how to copy data between CPU <=> GPU
  3. Learn by example how to avoid bad practices in OpenACC programming that could lead to potential performance issues
  4. Hands-on examples and exercises for accelerating scientific applications using PGI compiler
