Before teaching this lesson, the instructor should have:

  • Familiarity with uniprocessor caching, including that each cache line stores more data than is needed to satisfy a single memory request.
  • Enough comfort with C and OpenMP to understand the lab code. The code uses a parallel block and the function calls omp_get_thread_num and omp_get_num_threads (the functions are briefly defined).
  • Tested the given code on their platform to ensure that it shows the effect of false sharing. In particular, they may want to customize the size of image being given so that the measured part of the running time is stable for both the false sharing (i.e. small multiplier) and non-false sharing (i.e. large multiplier) versions of the code. Images of any size can be created by running the mandelbrot.c code from the "Race conditions" lesson of unit 4; simply modify the variables numRows and numCols (ideally keeping them the same).
  • Explored the Mandelbrot example from the "Race conditions" lesson (this is not strictly necessary, however, since the example just looks at the output).
